# Open source

Crucible is open source under the MIT license. This page covers the implications of that.

## License

MIT. See [LICENSE](https://github.com/10xdev4u-alt/crucible/blob/main/LICENSE).

In short:
- Use it commercially
- Modify it
- Distribute it
- Sublicense it
- Use it privately

The only requirement is to keep the copyright notice and license text in any copy.

## What "open source" means for Crucible

- **The source code is public** — anyone can read it, modify it, redistribute it
- **Anyone can contribute** — open issues, submit PRs, review code
- **The project is governed by maintainers** — see [MAINTAINERS](https://github.com/10xdev4u-alt/crucible/blob/main/docs/MAINTAINERS.md)
- **The roadmap is public** — see [ROADMAP](https://github.com/10xdev4u-alt/crucible/blob/main/docs/ROADMAP.md)
- **Releases are public** — see [CHANGELOG](https://github.com/10xdev4u-alt/crucible/blob/main/CHANGELOG.md)
- **Security issues are reported privately** — see [SECURITY](https://github.com/10xdev4u-alt/crucible/blob/main/SECURITY.md)

## What Crucible is NOT

- **Not open-core**: all features are in the open-source version
- **Not dual-license**: the code is MIT, period
- **Not source-available**: you can use, modify, redistribute without restriction
- **Not community-driven**: maintainers make the final call (but contributions are welcome)

## Trademark

The Crucible name and logo are not trademarked (yet). You can use them in your forks, plugins, and integrations. If we do trademark them in the future, we'll update this page.

## Why MIT?

MIT is the most permissive common license. It maximizes adoption by not imposing any restrictions beyond attribution. We want as many people as possible to use Crucible.

Alternatives we considered:
- **Apache 2.0**: similar to MIT but with an explicit patent grant. We chose MIT for simplicity.
- **GPL**: too restrictive, would block commercial use.
- **LGPL**: better than GPL but still complex.
- **BSL**: source-available but not open source. Not our model.
- **Elastic License v2**: same. Not our model.

## Governance

Crucible is maintained by a small team (see [MAINTAINERS](https://github.com/10xdev4u-alt/crucible/blob/main/docs/MAINTAINERS.md)). Decisions are made by lazy consensus. Major changes are discussed in issues or GitHub Discussions before implementation.
