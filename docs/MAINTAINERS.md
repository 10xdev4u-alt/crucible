## Becoming a maintainer

We're a small project. To become a maintainer:

1. Submit at least 5 merged PRs that demonstrate good judgment.
2. Show up to triage issues for at least 2 months.
3. Get an invite from an existing maintainer.
4. Sign the CLA (we'll set one up when we get there).

## Decision making

Most decisions are made by lazy consensus on PRs and issues.
Controversial changes are discussed in a dedicated issue or a
GitHub Discussion before implementation.
